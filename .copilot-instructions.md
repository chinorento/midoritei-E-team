# GitHub Copilot 指示書 - midoritei-E-team

このプロジェクトで作業する際は、必ず以下のルールに従ってください。

## 重要: プロジェクトガイドラインの遵守

**すべてのコード提案・生成は、プロジェクトルートの `README.md` に記載されているコーディングルールに従う必要があります。**

### コーディングルール（README.md より）

- **インデント統一**: タブやスペースの使い方を統一し、コードの階層を見やすくする
- **意味のある命名**: `price`、`user_name` など役割が明確な名前を使用（a、b などの一般的な名前は禁止）
- **コメント**: 複雑な処理や今後の変更箇所には日本語でコメントを記入
- **HTMLタグの対応**: `<p>` で始めたら `</p>` で必ず閉じる
- **id と class の使い分け**:
  - `id`: ページ内で1回のみ使用する要素
  - `class`: 複数回使用するデザイン要素
- **行の長さ**: 1行が長すぎないよう適切に改行を入れる
- **スタイルの統一**: スペースの空け方、改行ルールをプロジェクト全体で統一
- **CSS ファイル構成**:
  - `customer` フォルダ用: 1ファイル
  - `staff` フォルダ用: 1ファイル
  - 共通部分を流用し、個別部分はコメントで明示
- **JavaScript**: 別ファイルに分離
- **ファイル名**: 同じ機能のファイルは同じ名前を使用
- **変数命名**: 上記のルールに従う
- **エラー画面**: 作成必須

### API 仕様書の参照

ID や命名規則については、README.md に記載されている **API 仕様書（Ver. 2.1.0）** を参考にしてください。

## 注意事項

1. コード生成時は常にこのガイドラインを確認してください
2. 既存コードとの統一性を保ってください
3. 不明な点がある場合は、README.md の API 仕様書を参照してください
4. エラー処理は必ず実装してください
